CatholicTV — Cross-Platform Tracking Architecture

Goal: Connect individual viewer engagement across all 9 platforms into HubSpot for lead scoring + donation attribution  |  Updated: 2026-05-13

ℹ️ Architecture Assumption — Identity Provider
This diagram assumes Beacon User Management System (UMS) as the identity provider for all app platforms. If CatholicTV switches to an Open ID Connect (OIDC) provider (e.g. Microsoft Entra External ID), this architecture requires re-validation and Applicaster reconfiguration. The email-as-join-key approach may still apply, but the endpoint used to retrieve it will change. Raise with Sandro Fraga and Ran Meirman before proceeding with any identity provider migration.

🔧 CETDIGIT Middleware — What needs to be built
1
Google Analytics 4 data pull: Query GA4 Reporting Application Programming Interface on a schedule (daily/hourly), extract view events with user_id (email) dimension. Requires GA4 property access from CatholicTV.
2
HubSpot contact match: For each email in GA4, look up or upsert the corresponding HubSpot contact. Map video engagement fields (content watched, % complete, last viewed) to HubSpot contact properties.
3
Fallback handling (secondary): If any platform fails to resolve email via the Beacon Profiles Application Programming Interface, CETDIGIT can cross-reference a Beacon Registered Users export (token → email lookup). Export is currently manual UI only — no public Application Programming Interface. Feature request logged with Brightcove.
4
Orchestration: iPaaS recommended (Workato / Tray / Zapier Enterprise) or custom microservice. Brightcove Analytics Application Programming Interface does not push to HubSpot natively — pull-based architecture required.
