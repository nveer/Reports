Step-by-Step: Cloud Playout Channel Setup

Build the linear scheduled channel — the backbone of Newsday's programming

Phase 1 — Cloud Playout
  1. Navigate to Cloud Playout

    In Brightcove Studio, open the left nav and click Cloud Playout. Click Add New Channel.

  2. Name and Describe the Channel

    Enter channel details:

    • Name: e.g., Newsday-Linear-Channel
    • Description: "Newsday 24/7 news programming channel"
    • Optional: Image URL for EPG display
  3. Configure Schedule
    • Set Start Date/Time (allow 1–2 min for startup)
    • Enable Loop Playout Content for 24/7 operation
    • Leave End Date/Time blank if running indefinitely
  4. Enable Live Input

    Toggle Live Input: ON. This allows live breaking-news feeds to be injected into the scheduled program. Provide a Slate URL — this video plays when the live feed is unavailable (≈30 second delay).

  5. Configure Output Settings
    • Optimize Display for: Mobile & Web (or Smart TV if needed)
    • Output Region: Select closest to Newsday's encoder (US East = Virginia)
    • Output to: Brightcove Live Event — required for SSAI
    ⚠️ You must select "Brightcove Live Event" as output — not RTMP or S3 — to enable SSAI ad insertion.
  6. Enable Ad Insertion (SSAI)

    Check Ad Insertion. This enables SSAI for the channel. Mid-roll import toggle also appears here — enable it if using mid-roll ads on VOD content.

  7. Click "Add New" → Channel Created as DRAFT

    The channel moves through states: DRAFT → CREATING → SCHEDULED → RUNNING.

  8. Add Content to Program
    • Go to Manage Channel Program
    • Add VOD videos, playlists, and scheduled live events
    • Reorder content by drag-and-drop
    • Videos with ad cue points show a green dotted icon — these will trigger mid-roll ads
  9. Insert Ad Slots Between Shows

    To place an ad break between two content items in the schedule:

    • In the Manage Channel Program view, hover between two content items
    • Click the + Ad button that appears between them
    • Set the ad slot duration (minimum 30 seconds, maximum 10 minutes)
    • The slot appears in the schedule as a dedicated ad break — Cloud Playout will send a SCTE-35 marker at that point and SSAI will fill it with ads
    ℹ️ These between-show ad slots fire at their exact scheduled time with no delay — Cloud Playout knows the schedule in advance. This is different from live ad breaks, which carry an 18–30 sec buffer delay.
  10. Activate the Channel

    Click Activate. The channel begins streaming. Retrieve the RTP ingest point from Channel Settings for your live encoder. Note: when SSAI is enabled, Cloud Playout uses RTP (not RTMP) as the live input protocol — this is required to carry SCTE-35 ad markers through from the encoder.

Step-by-Step: Live Module (Breaking News Events)

Set up the Live event that Cloud Playout injects into — with SSAI enabled

Phase 2 — Live Streaming Module
ℹ️ No manual event creation needed. When you activate a Cloud Playout channel with SSAI enabled, Brightcove automatically creates a corresponding Live Event in the Live Module. You do not need to build it yourself. Phase 2 is about verifying and adjusting what was auto-created before going live.
  1. Open Live Module → Locate the Auto-Created Event

    After activating your Cloud Playout channel, navigate to Live module. You'll find a new Live Event that Cloud Playout created automatically. Click into it to review its settings.

  2. Verify Streaming Latency

    Cloud Playout auto-selects Regular latency (~30–45 sec delay). This is the correct default for a scheduled news channel.

    • Regular (default): ~30–45 second delay — recommended, matches HLS buffer inherent in the stream
    • Low Latency: ~5–10 second delay — available but at a higher cost
    💰 Cost consideration: Low Latency is billed at a higher rate than Regular. On a 24/7 channel this difference compounds significantly. Unless Newsday has a specific near-real-time requirement, Regular is the right choice. Confirm pricing with your Brightcove account team before changing this.
  3. Verify Ingest Profile

    Cloud Playout uses the account's default Live ingest profile for the auto-created event. Check that the default profile is appropriate for broadcast quality (multi-rendition, ABR).

    ⚠️ If you need a different ingest profile, update the default Live ingest profile in the Live Module settings before activating the Cloud Playout channel. Output settings on an already-created event cannot be modified after creation.
  4. Verify SSAI is Enabled

    Confirm the auto-created event has Server Side Ad Insertion (SSAI) enabled. This should be set automatically when Cloud Playout has Ad Insertion turned on.

    ⚠️ SSAI and output settings cannot be changed after event creation. If SSAI is missing, delete the event, correct the Cloud Playout channel configuration, and reactivate.
  5. Configure Fill Slate (in Cloud Playout)

    The fill slate is set within Cloud Playout channel settings — not in the Live Module event. In Cloud Playout, go to channel settings and select your pre-uploaded fill slate. This video plays whenever ad inventory goes unfilled during an ad break.

    ⚠️ Upload your fill slate to Admin → Live Settings before activating the channel. If no slate is configured, SSAI ad breaks may error or drop frames.
  6. Retrieve Both HLS Playback URLs

    From the Control Room or via the Live API, locate and save both output URLs — you'll need these for distribution:

    • SSAI URL (playlist_ssaiM.m3u8) — for Newsday's O&O website and apps
    • Clean HLS URL (playlist.m3u8) — for FAST channel partners (SCTE-35 intact)

    See the Publishing section below for full URL format details.

Step-by-Step: Ad Configuration (Pre-roll & Mid-roll)

Set up SSAI, configure ad breaks, and add cue points for mid-roll insertion

Phase 3A — SSAI / Ad Config in Admin
  1. Admin → Server-Side Ad Settings

    In Studio, go to Admin → Server-Side Ad Settings. Click Add Configuration.

  2. Create a Live Ad Configuration
    • Name: e.g., Newsday-SSAI-Live
    • Type: Live
    • Ad Server: Paste your VAST ad tag URL (DFP/GAM or other)
    • Configure beacons, targeting params as needed
  3. Admin → Live Settings → Upload Fill Slate

    Go to Admin → Live Settings. Upload at least one fill slate video. This plays during ad breaks with unsold/unfilled inventory to prevent a black screen.

  4. Save Ad Configuration

    Note the Ad Config ID — you'll reference it when publishing the player or building the SSAI playlist URL.

Phase 3B — Mid-Roll Cue Points on VOD Videos
  1. Open Media Module → Select a VOD Video

    Go to Media module and open any news segment video you want mid-rolls on.

  2. Add Ad Cue Points

    In the video detail, find the Cue Points section. Click Add Cue Point:

    • Type: Ad (not "Code" — Cloud Playout ignores code cue points)
    • Time: Enter the timestamp (e.g., 00:05:00 for a 5-minute mark)
    • Add multiple cue points per video for multiple ad breaks
    ℹ️ You can also set cue points programmatically via the CMS API for bulk updates across many videos.
  3. Save Cue Points → Verify in Cloud Playout

    After saving, return to Cloud Playout channel program. Videos with valid ad cue points display a green dotted icon. These will trigger mid-roll ad breaks at the exact cue point timestamps.

  4. Set Mid-Roll Duration (in Cloud Playout)

    In Cloud Playout channel settings, configure mid-roll ad duration:

    • Default: 30 seconds
    • Range: 30 seconds → 10 minutes
    • Can be changed anytime regardless of channel state (applies to next ad slot)
Phase 3C — Triggering Ad Breaks During Live (Control Room)
  1. Open Live Control Room

    During a live event, navigate to Live module → Control Room for your active event.

  2. Insert an Ad Break

    Click Go To Ad → Select the desired ad break duration → Click Insert. The fill slate plays for any unfilled inventory during that window.

    ⚠️ There will be a delay between clicking Insert and viewers seeing the ad. Due to the HLS segment buffer, expect 18–30 seconds from trigger to on-screen. Operators must account for this — trigger the break early, and plan a minimum 60-second ad window. See the Latency & Switching Expectations section at the bottom of this guide for the full breakdown.
  3. Manual vs. Automated Breaks

    For a news channel, ad breaks during live segments are triggered manually by a producer. During Cloud Playout VOD segments and between-show ad slots, breaks fire automatically at their scheduled time — no manual action needed and no buffer delay.

⚠️ RTMP outputs (e.g., social streaming to Facebook Live, YouTube) receive ad-free streams only. SSAI ads only appear in the Brightcove-delivered HLS/DASH stream.

    Use Brightcove's ad testing tools or a browser with network inspector to confirm:

    • Pre-roll fires on player load
    • Mid-roll fires at cue point timestamps during VOD segments
    • Fill slate appears when ad inventory is empty
    • Audio normalization prevents volume jump during ads
    • SCTE-35 markers visible in clean playlist.m3u8 via stream inspector

Operational Considerations

Answers to TC McCarthy's operational questions — day-to-day running of the platform

Real-World Operations — What to Expect

📼 Live-to-VOD Workflow

Status: No out-of-the-box automation with Static Entry Points (SEPs). Brightcove does not automatically clip and archive a live stream when using a recurring/persistent live event (SEP). Two workarounds are available:

  • Manual Clip: After the live segment ends, use the Live Module's built-in clipping tool (Video Cloud → Live → Clip) to trim the recorded stream into a VOD asset. Takes 2–5 minutes to process.
  • Automated Clipping via CRON/API: Use the Brightcove Live API POST /v1/clips on a scheduled job (e.g., every night at 2 AM) to auto-generate clips from a defined time window. Requires engineering effort to set up once, then runs hands-free.
⚠️ Newsday's engineering team may have an existing clipping/recording workflow that slots in cleanly. Confirm with TC before building a custom API job.

📋 Channel Assembly & Dynamic Playlist Updates

Cloud Playout supports real-time schedule changes while the channel is running. Operators can:

  • Drag-and-drop to reorder upcoming content in the schedule queue without interrupting the current segment
  • Add new VOD assets mid-run — they queue for the next available slot
  • Insert ad slots between any two segments on the fly
  • Override the current segment with a "Play Now" override (see Latency section below for delay expectations)
💡 Changes to the currently playing segment are not possible — only upcoming content can be reordered. Plan for at least one segment of buffer time when making last-minute changes.

📡 Live Event Handling — Persistent vs. On-The-Fly

Static Entry Points (SEP) give you a persistent, unchanging RTP ingest URL — the encoder connects once and the URL never rotates between activations. This is critical for Cloud Playout to reliably inject live feeds without manual reconfig every time. When SSAI is enabled, the live input uses RTP (not RTMP) to preserve SCTE-35 ad markers from the encoder.

Alternative — API-Driven Dynamic Entry Point: Newsday is evaluating using their Tyrion microservice to handle encoder bring-up via API — it pings Brightcove until the encoder is ready, then dynamically writes the RTMP endpoint back to Haivision automatically. This approach could eliminate manual reconfig entirely and provide maximum operational flexibility. Pending confirmation that the SEP conflict question is resolved before building this out.

ScenarioApproachLead Time
Scheduled live segment (news at 6PM) Pre-add SEP slot in Cloud Playout schedule; encoder connects ahead of time 15–30 min before air
Breaking news interrupt "Play Now" override in Cloud Playout to jump to live input; requires encoder already streaming ~30–60 sec to go live (see Latency)

⚠️ Cloud Playout requires the live input to be connected and streaming before the slot starts. If the encoder isn't sending signal, Cloud Playout will play the fill slate and advance to the next item after a short delay. Build a hot-standby encoder step into the ops runbook for time-sensitive content.

👥 Operational Model — Roles & Manual Intervention

A Cloud Playout + Live operation at this scale typically involves four distinct roles. Human oversight is required for live-to-live transitions — the platform does not auto-switch inputs without operator action.

  • Channel Director — owns the live schedule, makes Play Now calls, approves overrides. Primary interface: Cloud Playout Schedule UI.
  • Playout Operator — monitors channel health, manages encoder connections, responds to signal dropouts. Primary interface: Live Module dashboard + encoder software.
  • Content Editor — ingests VOD, tags ad cue points, manages metadata, queues upcoming content. Primary interface: Video Cloud Studio, Media module.
  • Ad Ops — manages SSAI configs, fill slate, and ad server (GAM/DFP) integration. Primary interface: Brightcove Admin → Live Settings, ad server.

⏱️ Latency & Switching Expectations

Ad timing behavior differs depending on where the ad originates — plan your runbooks accordingly:

ActionExpected DelayNotes
Pre-scheduled ad slots between shows ✅ Fires at exact scheduled time Cloud Playout knows the full schedule in advance and stitches ads precisely at the transition point — no buffer delay.
Mid-roll ads inside a VOD segment (ad-type cue points) ✅ Fires at exact cue point timestamp Also pre-compensatable — Cloud Playout knows the cue point position within the video and stitches accurately.
Live ad breaks (manually triggered during live feed) ⚠️ 18–30 sec delay after trigger HLS buffer delay cannot be pre-compensated for live signals. Operators must trigger ~30 sec early. Plan minimum 60-sec break windows.
End-to-end stream latency (encoder → viewer) 18–30 seconds HLS segment buffer (3 segments × 6–10 sec). Non-configurable in standard HLS.
"Play Now" override (jump to live input) 10–30 seconds CDN sync time; viewer experiences brief slate before live feed appears.
Pre-roll ad on stream load 0–3 seconds SSAI stitches before first segment; very fast.
Live feed dropout → slate fallback ~30 seconds Cloud Playout detects signal loss, plays fill slate, then advances schedule.
⚠️ Live ad breaks only: Instruct operators to trigger at least 30 seconds before the intended on-air time. For breaking news, scripted language like "we'll be right back" should precede the trigger by ~30 seconds.

📖 Lessons Learned — Common Pitfalls at Launch

  • Fill slate not uploaded before go-live: SSAI will error out or drop frames if there's no fill slate configured. Upload and test the slate before activating the channel. (Admin → Live Settings → Fill Slate)
  • Live encoder (RTP) connects too late: Cloud Playout requires the encoder to be streaming before the live slot starts. With SSAI enabled, the ingest is RTP — make sure your encoder is configured for RTP output, not RTMP. Build in a 15-minute warm-up window in your runbook.
  • Ad cue point type set to "Code" instead of "Ad": Code cue points are silently ignored by Cloud Playout. QA every video in the schedule to verify cue point type before launch.
  • MP4 rendition missing from ingest profile: Cloud Playout will skip or fail on any video without an MP4. Run a Media API audit before importing the content library.
  • SSAI not enabled at account level: This is a support request — it must be enabled by Brightcove before any SSAI config will work. Confirm with your Brightcove TAM before go-live week.
  • ✅ SCTE-35 passthrough confirmed: SCTE-35 markers DO survive the Fastly HLS URL (the clean, non-SSAI output). Confirmed April 2026. No third-party transcoder (e.g. Wowza) is required for FAST partner HLS delivery — use the Fastly URL directly for FAST channels, and the SSAI URL for O&O properties.
